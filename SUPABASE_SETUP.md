# Supabase Setup Guide for Noor Al Imaan Registration Form

## Database Configuration

### 1. Create the Registrations Table

Run the following SQL in your Supabase SQL Editor (found in the Supabase Dashboard):

```sql
-- Create registrations table for Noor Al Imaan registration form
-- Run the SQL from supabase_setup.sql file for complete setup
CREATE TABLE IF NOT EXISTS registrations (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    
    -- Personal Information (Afaan Oromo/Amharic)
    full_name_oromo TEXT,
    age_oromo INTEGER,
    city_oromo TEXT,
    gender_oromo TEXT,
    village_oromo TEXT,
    mosque_oromo TEXT,
    phone_oromo TEXT,
    
    -- Personal Information (English/Amharic)
    full_name_english TEXT,
    age_english INTEGER,
    city_english TEXT,
    gender_english TEXT,
    village_english TEXT,
    mosque_english TEXT,
    phone_english TEXT,
    
    -- Participation Options
    participation_idea BOOLEAN DEFAULT FALSE,
    participation_money BOOLEAN DEFAULT FALSE,
    participation_material BOOLEAN DEFAULT FALSE,
    participation_provision BOOLEAN DEFAULT FALSE,
    participation_all BOOLEAN DEFAULT FALSE,
    
    -- Metadata
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Add indexes for better performance
CREATE INDEX idx_registrations_created_at ON registrations(created_at);
CREATE INDEX idx_registrations_city ON registrations(city);
CREATE INDEX idx_registrations_district ON registrations(district);

-- Add Row Level Security (RLS) policies
ALTER TABLE registrations ENABLE ROW LEVEL SECURITY;

-- Allow anyone to insert data (for public registration form)
CREATE POLICY "Allow public insert" ON registrations
    FOR INSERT WITH CHECK (true);

-- Allow anyone to read their own data (optional)
CREATE POLICY "Allow public read" ON registrations
    FOR SELECT USING (true);
```

## Troubleshooting

### Common Issues:
1. **CORS Errors**: Make sure your domain is added to Supabase CORS settings
2. **Permission Denied**: Check RLS policies are correctly configured
3. **Connection Failed**: Verify project URL and key are correct

### Debug Steps:
1. Open browser developer tools
2. Check Network tab for API requests
3. Review Console for error messages
4. Verify data in Supabase dashboard
